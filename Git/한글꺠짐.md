# 한글 깨짐 오류

1. 발생일시 : 2022.11.18
2. 장소 : 회사 컴, 내 맥북 로컬
3. 무엇 : TIL 레포로 한글 디렉토리 혹은 한글 파일 생성중 발생
4. 오류 출력: 
```bash
❯ git status
현재 브랜치 main
커밋하도록 정하지 않은 변경 사항:
  (무엇을 커밋할지 바꾸려면 "git add <파일>..."을 사용하십시오)
  (use "git restore <file>..." to discard changes in working directory)
        수정함:        .gitignore

추적하지 않는 파일:
  (커밋할 사항에 포함하려면 "git add <파일>..."을 사용하십시오)
        "Algorithms/\354\225\214\352\263\240\353\246\254\354\246\230\355\215\274\354\246\22068/"

커밋할 변경 사항을 추가하지 않았습니다 ("git add" 및/또는 "git commit -a"를
사용하십시오)
```
5. 발견 과정: `git status` 명령어로 상태 확인중 한글 문자가 표현 되지 않음 발견
6. 해결 방법: 
 1) git config 명령어에서 --global 매개변수의 서브 키 core.quotepath 값을 false로 설정
7. 발생원인과 접근방법 : 일반적이지 않은 문자는 escaping 문자로 인식하여 \를 붙여주기 때문에 이에 따라 이 설정을 꺼주면 정상적으로 한글이 나타나는걸 확인 할 수 있음
